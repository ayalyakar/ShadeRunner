# DESIGN - Writing

## Decision

**DECIDED:** Three-layer writing pipeline:

1. **Hand-written core** — main plot, campaign arc beats, canonical NPC personalities, signature lines.
2. **Hand-written core + templated procedural** — ambient dialogue, news headlines, BBS chatter, minor NPC lines use templates filled at runtime.
3. **LLM-assisted authoring (at development time)** — LLMs help draft variants that a human edits and approves; all output is stored as static data in the repo.

Explicitly excluded: **runtime LLM generation**. See `Technology/DESIGN - AI Tooling.md`.

## Pipeline

```
  idea  →  human seed prompt  →  LLM draft  →  human edit  →  approved content.json
                                                                   │
                                                                   ▼
                                                              runtime reads data only
```

- Nothing calls an LLM at runtime.
- Every line shipped was reviewed by a human (even if generated by an LLM first).
- Templates use variables filled from sim state (e.g. `{{contact.name}} mentioned {{last_event.rumor}}`).

## Content types

### Main plot & arc dialogue
- Hand-written, full fidelity.
- Branching, choice-tagged, with reaction variants per player state.
- The most expensive content; budget carefully.

### Side runs
- Template-driven. Run "scripts" use placeholders for target / complication / location.
- Each template has multiple dialogue variants per Johnson archetype.
- LLM helps produce variants fast; human edits ensure tone consistency.

### Ambient content
- **News headlines** — templated, with per-event hooks (corp merger, gang raid, weather).
- **BBS / Shadowland threads** — templated with procedurally-generated thread structures.
- **NPC idle lines** — short barks triggered by context (time, district, weather, PC rep).
- **Trid ads** — era-appropriate, per corp and district.
- **Email inbox filler** — adds texture to commlink.

### Contacts
- Each major contact has a voice guide (personality, vocabulary, tics).
- Scripted scenes for trust milestones.
- Procedural intermediate banter.

### Emails / messages
- Templates with variable slots; per-sender voice profiles.

## Tone management

- The game has **no imposed tone** (see `Design/DESIGN - Vision.md`).
- Each NPC / situation has its own register; the player's choices shape overall feel.
- Tone tags on content let the engine pick appropriate variants per context.

## Voice acting

- Full voice acting is **planned** long-term (see `Presentation/DESIGN - Audio.md`) but huge effort.
- Solo approach: AI TTS for ambient and placeholder; hero lines may eventually get human VO (or not).
- All dialogue has subtitles.
- Voice casts can be inconsistent / swapped as long as character voice profiles stay consistent per NPC.

## Localization

- English first.
- Localizability baked in: all shipped text in keyed resource files.
- No localization commitment beyond English in the near term.

## Open questions

- LLM choice for drafting (local vs. hosted, quality vs. control).
- How much ambient variation is needed before repetition fatigue.
- Writing style guide — clear voice per POV, register per venue.
- Authorial voice across decades — do early-era news headlines sound different from late-era?
